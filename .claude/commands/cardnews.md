# /cardnews — 유튜브·기사·논문 → 닷커넥터 카드뉴스

## 사용법
/cardnews <YouTube URL | 웹 URL | PDF 경로 | 로컬 md> [옵션]

## 실행

`$ARGUMENTS`가 비어 있으면 입력(URL/PDF)을 먼저 물어보세요.

```bash
py -X utf8 scripts/cardnews.py <입력> --cards 8
```

## 설명
입력을 분석해 1080x1350(4:5) 카드뉴스 PNG 세트를 생성합니다.
출력: 바탕화면 `cardnews/<날짜>-<슬러그>/card-01.png ...` + `cards.json`(재편집용).

**디자인**: 아이보리 배경 + 닷커넥터 로고 + 날짜, 2줄 헤드라인(둘째 줄 노란 형광),
라운드 이미지, 2줄 본문(존칭체), 출처·페이지 푸터. 레퍼런스는 2026-07-19 수신
카드뉴스 스타일을 닷커넥터 브랜딩으로 번안한 것.

**카드 이미지 소스**:
- YouTube → 실제 영상 프레임 (yt-dlp 최저화질 + OpenCV, 인트로·아웃트로 10% 제외 균등).
  푸터에 실제 프레임 타임스탬프("화면 MM:SS") 자동 표기 — 시각을 지어내지 않음
- 논문·기사 → **Gemini 이미지 생성**(`gemini-2.5-flash-image`, 16:9, 한국 사무실·교실
  포토리얼 톤). 생성 실패 시 노란 텍스트 패널 카드로 자동 대체
- `GEMINI_API_KEY`는 `.env`에서 자동 로드

## 옵션
- `--cards N` : 카드 수 (기본 8, 표지·마무리 포함)
- `--dry-run` : 카피 JSON만 출력 (렌더 없음 — 카피 검토용)
- `--no-imggen` : 이미지 생성 생략 (전부 텍스트 패널)
- `--out DIR` : 출력 폴더 지정
- `--model M` : 카피 모델 (기본 gemini-2.5-flash)

## 실행 후 QA (필수 — 생성 직후 항상 점검)
1. **카피 사실성**: cards.json의 수치·주장을 원문과 대조. 원문에 없는 내용이 있으면
   cards.json 수정 후 재렌더(아래 재렌더 방법)
2. **헤드라인 이어읽기**: headline_top + highlight가 한 문장으로 읽히는지
3. **문체**: 존칭체(~합니다) 유지, 금지 표현(S1) 혼입 여부
4. **이미지-내용 관련성**: 각 카드 PNG를 열어 이미지가 본문과 어긋나지 않는지 육안 확인.
   생성 이미지에 텍스트·왜곡 손가락 등 아티팩트가 있으면 해당 카드만 재생성
5. **글자 넘침**: 헤드라인·본문이 잘리거나 줄바꿈이 어색한 카드 확인

## 재렌더 (카피 수정 후)
cards.json을 수정한 뒤 전체 재실행 없이 render/ 폴더의 HTML을 직접 수정하거나,
같은 입력으로 재실행(--out 동일 폴더). 프레임·생성 이미지는 render/에 남아 재사용 가능.

## 알려진 특성
- 유튜브 자막 없는 영상은 description으로 폴백 → 카피가 얕아질 수 있음 (본문 200자
  미만이면 환각 방지를 위해 생성 자체를 중단)
- 이미지 생성은 카드당 1회 API 호출 — 8장이면 이미지 8회 + 카피 1회
- 프레임은 내용 매칭이 아닌 균등 분포 — 타임스탬프는 실제지만 장면-내용 정합은 우연
