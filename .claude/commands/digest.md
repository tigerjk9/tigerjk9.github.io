# /digest — 주간 다이제스트 포스트 자동 생성

## 사용법
/digest [옵션]

## 설명
지난 7일간 `_posts/`에 올라온 포스트를 모아 주간 다이제스트 포스트를 생성합니다.
주제별 섹션으로 재구성하고, 각 글마다 "왜 읽을 가치가 있는지" 한 줄 코멘트와 퍼머링크를 붙입니다.
GEMINI_API_KEY는 프로젝트 루트의 `.env` 파일에서 자동으로 읽습니다.

## 실행

아래 명령어를 실행하세요 (`$ARGUMENTS`에 사용자 옵션이 있으면 그대로 전달):

```bash
py scripts/weekly_digest.py $ARGUMENTS
```

옵션: `--dry-run`(출력만) / `--no-push`(로컬 저장만) / `--days N`(기간 변경) / `--date YYYY-MM-DD`(기준일)

## 실행 후 반드시 (후처리 QA)

생성된 `_posts/YYYY-MM-DD-weekly-digest.md`를 열어 공통 QA 체크리스트를 적용하세요:

1. front matter 직후 잔류 코드펜스 확인
2. **모든 링크가 실제 존재하는 퍼머링크인지 확인** — 목록의 대상 포스트 permalink와 대조 (Gemini가 링크를 지어낼 수 있음)
3. S1 금지 표현 혼입 확인 (`~를 통해`, `~을 넘어`, `결론적으로`, 콜론 헤딩, 이모지) → 교정
4. 대상 포스트 누락 확인 — 스크립트 로그의 대상 목록과 본문 불릿 수 대조
5. 존칭 어미 혼입 확인 (단정체 유지)

교정 후 별도 커밋하세요.

## 동작 특성
- 대상 기간 포스트가 3편 미만이면 생성하지 않음
- `주간다이제스트` 태그 포스트는 수집에서 제외 (자기 참조 방지)
- 파일명 `YYYY-MM-DD-weekly-digest.md`, 퍼머링크 `/post/weekly-digest-YYYY-MM-DD/`
- 카테고리 `[다이제스트]` — 카테고리 페이지 카운트가 1 증가하는 것은 정상
