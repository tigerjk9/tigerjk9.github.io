## [슬라이드 1] AI 시대 코딩, 실수해도 괜찮아!

**핵심 내용**
- AI의 코딩 지원과 동시에 발생하는 오류 관리의 어려움
- 교실 코딩 교육 현장에서 '실수'에 대한 부담감
- Git/GitHub이 제공하는 '안전망'의 필요성 강조

**강의 원고**
> 선생님들, 안녕하세요! 요즘 AI 기술, 특히 챗GPT나 코파일럿 같은 도구들이 코딩 교육 현장에 가져온 변화를 실감하고 계실 겁니다. AI가 코드를 뚝딱 만들어주니 학생들도 교사들도 코딩 접근성이 훨씬 높아졌죠. 그런데 말입니다, AI가 만들어준 코드가 항상 완벽하던가요? 때로는 이상한 코드 때문에 프로젝트 전체가 꼬이거나, 어디서부터 손대야 할지 막막한 경험, 다들 한 번쯤 있으실 겁니다. 학생들은 또 어떻고요. 코딩하다가 실수라도 할까 봐 지레 겁먹고 시도조차 하지 않으려는 모습도 보이고요.
>
> 오늘 우리는 이런 코딩 교육 현장의 고민을 해결해 줄 아주 강력한 도구, 바로 **Git**과 **GitHub**에 대해 이야기해 보려고 합니다. 코딩 실수를 **'휙' 두 번 만에 되돌리고**, AI와도 효과적으로 협업하며, 우리 학생들을 진정한 디지털 탐험가로 만들어 줄 이 도구들을 함께 알아가 보시죠. 실수는 성장의 기회이지, 두려움의 대상이 아닙니다!

## [슬라이드 2] 왜 지금 Git/GitHub인가? (1) AI 시대 코딩 교육의 필수 안전망

**핵심 내용**
- AI의 빠른 코드 생성 속도와 비례하는 오류 발생 가능성
- 코딩 프로젝트의 '타임머신' 역할을 하는 버전 관리 시스템
- 시행착오를 두려워하지 않는 학습 환경 조성

**강의 원고**
> 과거에는 코딩을 하려면 모든 코드를 직접 입력해야 했습니다. 한 글자라도 틀리면 오류가 나고요. 그래서 실수를 줄이는 것이 중요했죠. 하지만 지금은 AI가 수십, 수백 줄의 코드를 순식간에 생성해 줍니다. 매우 편리하지만, 여기에 함정이 있습니다. AI가 만들어준 코드가 잘못되었을 때, 또는 내 의도와 다르게 작동할 때, 우리는 어디서부터 무엇을 고쳐야 할지 혼란에 빠지기 쉽습니다. 마치 복잡한 미로에 갇힌 기분이랄까요.
>
> 이럴 때 필요한 것이 바로 코딩 프로젝트의 **'타임머신'**입니다. 마치 게임에서 중요한 순간마다 '세이브 포인트'를 만들어두는 것처럼, 내가 코드를 변경할 때마다 그 상태를 저장해두는 거죠. 그래야 언제든 문제가 생기면 안전하게 이전 상태로 **되돌릴 수 있습니다.** AI가 코드를 망가뜨려도 '휙' 두 번이면 되돌릴 수 있다는 말이 바로 여기서 나옵니다. Git과 GitHub은 이러한 **버전 관리**를 가능하게 하여, 우리와 학생들이 코딩 과정에서 시행착오를 두려워하지 않고 과감하게 도전할 수 있는 안전한 학습 환경을 제공합니다.

## [슬라이드 3] 왜 지금 Git/GitHub인가? (2) 협력과 공유의 가치를 높이는 도구

**핵심 내용**
- 교실 프로젝트 및 교사 학습 공동체 협업의 비효율성 문제
- 수업 자료, 평가 문항 등 교육 콘텐츠 공유 및 버전 관리의 어려움
- Git/GitHub이 제공하는 효율적인 협업 및 공유 플랫폼

**강의 원고**
> 코딩 교육이 발전하면서 학생들의 협력 프로젝트가 점점 늘어나고 있습니다. "우리 팀 프로젝트가 최신 버전이 맞나요?", "누가 뭘 수정했는지 모르겠어요", "내 코드가 다른 친구 코드를 덮어썼어요!" 이런 이야기, 교실에서 흔히 듣지 않으시던가요? 비단 학생들만의 이야기는 아닙니다. 교사 학습 공동체에서 함께 수업 자료를 개발하거나, 동료 교사들과 평가 문항을 공유할 때도 최신 버전을 찾기 어렵거나, 누가 어떤 피드백을 반영했는지 혼란스러울 때가 많습니다.
>
> **Git**과 **GitHub**은 이러한 협업과 공유의 문제를 해결하는 데 탁월한 도구입니다. 누가 언제 무엇을 변경했는지 모든 기록을 투명하게 관리하고, 여러 사람이 동시에 작업해도 충돌을 최소화하며, 언제든 가장 최신 버전을 쉽게 공유하고 내려받을 수 있도록 돕습니다. 마치 공동으로 글을 쓰는 '클라우드 공유 노트'와 같습니다. 우리 학생들에게는 미래 사회에 필수적인 협업 능력을 길러주고, 교사들에게는 교육 콘텐츠 개발의 효율성을 높여줄 핵심 도구가 될 것입니다.

## [슬라이드 4] Git이란 무엇인가? – 코딩 프로젝트의 '세이브 포인트'

**핵심 내용**
- **Git**: 코드 변경 이력을 기록하고 관리하는 **버전 관리 시스템**
- **저장소(Repository)**: 프로젝트의 모든 파일과 변경 이력이 저장되는 공간
- **커밋(Commit)**: 프로젝트의 특정 시점 상태를 저장하는 '세이브 포인트'
- **히스토리(History)**: 모든 커밋이 기록된 변경 이력

**강의 원고**
> 자, 그럼 본격적으로 Git이 무엇인지부터 살펴보겠습니다. **Git**은 여러분의 코딩 프로젝트가 마치 게임처럼 '세이브 포인트'를 가질 수 있도록 돕는 시스템입니다. 코드를 변경할 때마다 그 순간의 상태를 사진 찍듯이 저장해 두는 거죠. 이렇게 저장된 단위 하나하나를 **'커밋'**이라고 부릅니다.
>
> 이 모든 커밋, 즉 세이브 포인트들이 모여서 하나의 긴 **'히스토리'**, 즉 변경 이력이라는 시간 선을 만듭니다. 그리고 이 모든 파일과 히스토리가 담겨 있는 곳을 우리는 **'저장소'**, 영어로는 **레포지토리(Repository)**라고 합니다. 내 컴퓨터에 있는 프로젝트 폴더 하나하나가 나만의 Git 저장소가 될 수 있습니다. 만약 프로젝트에 문제가 생겨도 걱정 없습니다. 타임머신처럼 원하는 시점의 커밋으로 돌아가서 다시 시작할 수 있으니까요. 정말 든든한 안전망이죠.

## [슬라이드 5] Git 핵심 개념 (1) – 커밋 메시지: 무엇을, 왜 변경했는지 기록하기

**핵심 내용**
- **커밋 메시지**: 변경 내용을 요약하고 그 이유를 설명하는 기록
- 명확한 커밋 메시지의 중요성: 자기 성찰 및 협업의 효율성 증대
- 블로그 사례: "공유누리 API POST 방식 발견"과 같은 기록의 가치

**강의 원고**
> 앞서 커밋이 '세이브 포인트'라고 말씀드렸죠? 이 세이브 포인트를 만들 때는 '무엇을, 왜 변경했는지'를 기록하는 **'커밋 메시지'**를 남깁니다. 단순한 '저장' 버튼과는 다릅니다. 이 메시지는 나중에 내가 봐도, 다른 사람이 봐도 이 커밋에서 어떤 변화가 있었는지 바로 알 수 있도록 돕는 일종의 '메모'입니다.
>
> 여러분, 혹시 블로그 포스트 "공공 공유자원 검색 플랫폼 개발 후기" 기억하시나요? 거기 보면 '공유누리 OPEN API의 경우, 현재 POST 메서드만 지원하고 있다는 것을 공식 문서가 아닌 홈페이지 문의 게시판을 통해 뒤늦게 알게 되었다'는 시행착오 경험이 나옵니다. 만약 이때 이 내용을 커밋 메시지로 남겼다면 어땠을까요? 나중에 다시 코드를 볼 때, 왜 이렇게 코드를 짰는지, 어떤 어려움이 있었는지 바로 파악할 수 있었을 겁니다. 이런 기록은 나 자신과의 소통이자, 미래의 나와 협업하는 동료들을 위한 중요한 이정표가 됩니다. 학생들에게도 커밋 메시지를 통해 자신의 코드 변경 과정을 성찰하고 설명하는 연습을 시킬 수 있죠.

## [슬라이드 6] Git 핵심 개념 (2) – 브랜치: 안전하게 실험하는 '평행 우주'

**핵심 내용**
- **브랜치(Branch)**: 메인 코드 흐름에서 분기하여 독립적으로 작업하는 공간
- 새로운 기능 개발이나 문제 해결을 메인 코드에 영향 없이 진행
- 안전한 실험 환경을 제공하여 창의적인 코드 작성 유도

**강의 원고**
> 다음으로 소개할 Git의 강력한 기능은 바로 **'브랜치'**입니다. 브랜치는 마치 소설에서 '만약 이랬다면?' 하고 이야기를 다른 방향으로 이끌어가는 '평행 우주'와 같습니다. 여러분의 코딩 프로젝트가 하나의 메인 이야기라고 할 때, 브랜치는 이 메인 이야기에 영향을 주지 않고 새로운 기능을 개발하거나 어떤 문제를 해결해 보는 독립적인 작업 공간을 만들어줍니다.
>
> 예를 들어, 여러분이 주요 수업 자료(메인 브랜치)를 안정적으로 운영하고 있다고 가정해 봅시다. 그런데 새로운 인공지능 활용 수업 활동(새로운 기능)을 추가하고 싶어요. 이때 메인 자료를 건드리지 않고, 새로운 브랜치를 만들어서 마음껏 코딩하고 실험해 볼 수 있습니다. 만약 잘 완성되면 메인 자료에 합치고(Merge), 아니다 싶으면 그 브랜치를 버리면 됩니다. 메인 코드가 망가질 걱정 없이 얼마든지 자유롭게 시도하고, 실험할 수 있죠. 학생들에게도 이러한 브랜치 활용을 통해 '실패해도 괜찮으니 마음껏 시도해 보라'는 메시지를 줄 수 있습니다.

## [슬라이드 7] GitHub이란 무엇인가? – 코드를 위한 '클라우드 공유 노트'

**핵심 내용**
- **GitHub**: Git 저장소를 호스팅하는 웹 기반 플랫폼
- **원격 저장소(Remote Repository)**: 온라인에 코드를 저장하고 공유하는 공간
- 협업의 중심지 역할: 코드 공유, 백업, 다른 사람의 코드 탐색
- Git과 GitHub의 역할 구분 명확화

**강의 원고**
> Git이 내 컴퓨터 안에서 '세이브 포인트'를 만들고 '히스토리'를 관리하는 시스템이라면, **GitHub**은 이 Git 저장소들을 전 세계 개발자들과 공유하고 협업할 수 있도록 돕는 웹 서비스입니다. 쉽게 말해, 구글 드라이브나 네이버 MYBOX에 여러분의 문서나 사진을 올리듯, Git으로 관리하는 코드를 웹 공간에 올려두는 **'클라우드 공유 노트'**라고 생각하시면 됩니다.
>
> 내 코드가 온라인에 안전하게 백업되니 컴퓨터 고장 걱정 없이 안심할 수 있고, 언제 어디서든 인터넷만 연결되면 내 코드에 접근할 수 있습니다. 무엇보다 **GitHub**의 가장 큰 장점은 바로 **협업**입니다. 여러 사람이 하나의 프로젝트에 참여하여 코드를 함께 수정하고, 서로의 코드를 검토하며, 심지어 다른 사람들의 공개된 코드 저장소를 구경하고 가져와 배울 수도 있습니다. Git이 내 코드를 지키는 개인 비서라면, GitHub은 전 세계 개발자들과 소통하는 커뮤니티이자 나의 코드를 전시하는 갤러리 역할을 합니다.

## [슬라이드 8] 터미널 없이 Git/GitHub 시작하기 – GitHub Desktop

**핵심 내용**
- 코딩 경험이 없는 교사에게 **GitHub Desktop**의 필요성 강조
- 명령어 입력 없이 GUI(그래픽 사용자 인터페이스)로 모든 Git/GitHub 작업 수행
- 초보자도 쉽게 접근할 수 있는 직관적인 사용 환경

**강의 원고**
> Git과 GitHub에 대한 설명을 듣다 보니 '명령어'나 '터미널' 같은 용어에 부담을 느끼셨을 수도 있습니다. 하지만 걱정하지 마세요! 오늘 연수의 핵심 중 하나는 바로 **GitHub Desktop**입니다. 이것은 마치 복잡한 컴퓨터 게임을 마우스 클릭 몇 번으로 쉽게 즐길 수 있도록 돕는 그래픽 인터페이스와 같습니다.
>
> **GitHub Desktop**은 Git의 모든 강력한 기능을 **터미널 명령어 없이** 마우스 클릭과 드래그앤드롭만으로 수행할 수 있게 해줍니다. 저장소를 만들고, 파일을 추가하고, 커밋하고, GitHub에 올리는 모든 과정이 직관적인 화면에서 이루어지죠. 코딩 경험이 전혀 없는 교사분들도 이 도구만 있으면 Git과 GitHub을 손쉽게 활용할 수 있습니다. 복잡한 명령어를 외울 필요 없이, '휙'하고 마우스로 조작하면 모든 것이 가능해집니다.

## [슬라이드 9] GitHub Desktop으로 실습! – 저장소 만들고 첫 커밋하기

**핵심 내용**
- GitHub Desktop을 활용한 **로컬 저장소 생성**
- 파일 추가 및 변경 사항 확인 과정
- 첫 **커밋** 생성 및 **GitHub에 푸시(Push)**하여 원격 저장소에 반영

**강의 원고**
> 자, 그럼 이제 GitHub Desktop으로 직접 Git과 GitHub을 경험해 볼 시간입니다. 실제로 화면을 보면서 따라 해 본다고 상상해 보세요. 먼저 GitHub Desktop을 실행하고 '새 저장소 만들기' 버튼을 누릅니다. 프로젝트 이름을 정하고, 내 컴퓨터 어디에 저장할지 폴더를 지정하기만 하면 끝! 아주 간단하게 나만의 **로컬 저장소**가 만들어집니다.
>
> 이제 이 폴더 안에 여러분의 코딩 파일이나 수업 자료 파일을 넣어보세요. GitHub Desktop 화면에 어떤 파일이 추가되었는지, 어떤 내용이 변경되었는지 바로 나타날 겁니다. 마치 '변경 사항 감지기' 같죠. 변경된 내용을 확인한 후, '여기에 커밋 메시지를 작성하세요'라는 칸에 "첫 번째 프로젝트 시작" 같은 메시지를 입력하고 '커밋' 버튼을 누르면 첫 번째 **커밋**, 즉 첫 번째 세이브 포인트가 만들어지는 겁니다. 이제 이 커밋을 GitHub 웹사이트, 즉 **원격 저장소**에 올리고 싶다면 'Push origin' 버튼만 누르면 됩니다. 여러분의 코드가 전 세계와 연결되는 순간이죠!

## [슬라이드 10] 실수해도 괜찮아! – 코드 '되돌리기' 완전 정복 (1): Revert

**핵심 내용**
- **Revert**: 특정 커밋의 변경 사항을 되돌리는 새로운 커밋을 생성하는 기능
- 과거 기록을 삭제하지 않고, '되돌림' 기록을 남기는 안전한 복구 방식
- 코딩 실수를 두려워하지 않고 과감하게 도전할 수 있는 기반 마련

**강의 원고**
> 코딩을 하다 보면 누구나 실수를 합니다. 아니, 실수는 코딩 학습의 자연스러운 과정이죠. 중요한 것은 실수를 '어떻게 관리하고 되돌리는가'입니다. 여기서 Git의 강력한 기능인 **'Revert'**가 빛을 발합니다. Revert는 마치 잘못 제출한 숙제를 '이 숙제는 틀렸으니 없애주세요'라고 요청하면, 그 내용을 다시 기록해서 수정본을 제출하는 것과 같습니다.
>
> 즉, 문제가 발생한 특정 커밋을 찾아 '이 커밋에서 변경된 내용을 없애는 새로운 커밋을 만들어줘'라고 명령하는 거죠. 이렇게 하면 과거의 잘못된 기록을 아예 지워버리는 것이 아니라, '이때의 실수를 되돌렸음'이라는 새로운 기록을 남기기 때문에 훨씬 더 안전하고 투명하게 프로젝트 이력을 관리할 수 있습니다. 학생들에게도 실수를 지우기보다 '실수를 수정하는 과정'을 기록으로 남기는 학습 습관을 길러줄 수 있습니다.

## [슬라이드 11] 실수해도 괜찮아! – 코드 '되돌리기' 완전 정복 (2): 히스토리 탐색

**핵심 내용**
- **히스토리** 탐색: 프로젝트의 모든 변경 이력을 시각적으로 확인
- 원하는 과거 시점으로 돌아가 프로젝트 상태 복구 또는 분기 (브랜치)
- 블로그 사례: API 명세서 불확실성과 같은 개발 과정의 불확실성을 Git으로 관리

**강의 원고**
> Revert가 특정 커밋의 실수를 되돌리는 것이라면, Git의 **'히스토리'**는 우리에게 훨씬 더 강력한 '타임머신' 능력을 부여합니다. GitHub Desktop에는 프로젝트의 모든 커밋 이력이 시간 순서대로 그래프처럼 표시됩니다. 이 히스토리를 통해 우리는 마치 과거로 돌아가 사진첩을 넘기듯, 어느 시점에서 어떤 코드가 어떻게 변경되었는지 한눈에 확인할 수 있습니다.
>
> 블로그에서 소개된 "공유누리 Open API 활용" 사례를 보면, 'API 명세서를 100% 신뢰하지 말라'는 교훈이 나옵니다. 실제 API가 명세서와 다르게 작동하는 경우가 있기 때문이죠. 만약 이런 상황에서 Git의 히스토리를 활용했다면, 특정 API 연동 코드를 여러 방식으로 시도하면서 각 시도마다 커밋을 남기고, 문제가 발생했을 때 언제든 이전의 안정적인 상태로 쉽게 돌아갈 수 있었을 겁니다. 이렇게 히스토리를 잘 활용하면 프로젝트가 아무리 복잡해지고 예측 불가능한 변수가 생겨도, 우리는 안전하게 코드를 관리하고 복구할 수 있습니다.

## [슬라이드 12] AI와 협업하는 코딩 워크플로우 (1) – GitHub Copilot & 자동 커밋

**핵심 내용**
- **GitHub Copilot**: AI 기반 코드 자동 완성 및 생성 도구
- **VSCode Source Control** 패널과의 연동을 통한 Git/GitHub 작업
- AI의 도움을 받되, 교사/학생이 최종 제어권을 가지는 워크플로우

**강의 원고**
> AI 시대의 코딩은 혼자 하는 것이 아닙니다. 이제 AI가 우리의 든든한 조력자가 되어줍니다. 특히 **GitHub Copilot**은 우리가 코드를 작성하는 동안 옆에서 실시간으로 코드를 제안하고, 심지어는 긴 코드 블록까지 통째로 만들어주는 인공지능 프로그래머입니다.
>
> 그리고 이러한 AI 생성 코드를 효율적으로 관리하는 방법이 바로 **VSCode의 Source Control 패널**을 활용하는 것입니다. Visual Studio Code에서 코드를 작성하고 수정하면, Source Control 패널에서 변경된 파일들을 실시간으로 보여줍니다. 여기서 AI가 제안하는 커밋 메시지를 활용하여 변경 사항을 쉽게 커밋하고, GitHub에 푸시할 수 있습니다. AI가 코드를 만들고, AI가 커밋 메시지를 제안해주지만, 최종적으로 '커밋' 버튼을 누르고 '푸시'하는 것은 바로 우리입니다. AI의 도움을 최대한 받되, 중요한 결정과 제어권은 우리가 가지는 것이죠.

## [슬라이드 13] AI와 협업하는 코딩 워크플로우 (2) – 안전한 AI 프로젝트 관리

**핵심 내용**
- **API 키** 등 민감 정보의 외부 노출 위험성
- **.gitignore** 파일을 활용한 민감 정보 보호
- AI 서비스 활용 시 보안 규칙 준수의 중요성 및 교육적 함의

**강의 원고**
> AI 서비스를 활용한 코딩 프로젝트를 진행하다 보면, **API 키**와 같은 민감한 정보들을 사용해야 할 때가 많습니다. '랜드마크 탐험관 V2' 프로젝트에서도 구글 Gemini API 키를 활용했듯이 말이죠. 이러한 API 키는 마치 우리 집 열쇠와 같아서 외부에 노출되면 심각한 보안 문제가 발생할 수 있습니다.
>
> 그런데 GitHub에 프로젝트 코드를 올릴 때, 실수로 이런 민감한 정보가 포함된 파일을 함께 올려버리면 어떻게 될까요? 전 세계 사람들이 내 API 키를 볼 수 있게 되는 겁니다. 이를 방지하기 위해 우리는 **'.gitignore'**라는 특별한 파일을 사용합니다. '.gitignore' 파일에 특정 파일이나 폴더 이름을 적어두면, Git은 그 파일들을 Git 저장소에 포함시키지 않습니다. GitHub에 푸시될 때도 마찬가지고요. 이를 통해 중요한 개인 정보나 API 키를 안전하게 보호할 수 있습니다. 학생들에게도 이러한 보안 규칙의 중요성을 가르치는 것은 AI 시대를 살아가는 필수적인 디지털 리터러시 교육이 될 것입니다.

## [슬라이드 14] 함께 성장하는 프로젝트 – Fork, PR, 그리고 충돌 해결

**핵심 내용**
- **Fork**: 다른 사람의 저장소를 내 GitHub 계정으로 복사하여 독립적으로 작업
- **Pull Request (PR)**: Fork한 저장소에서 변경한 내용을 원본 저장소에 반영 요청
- **충돌 해결**: 여러 사람이 동시에 작업할 때 발생하는 충돌을 해결하는 과정

**강의 원고**
> Git과 GitHub의 진정한 가치는 바로 **협업**에 있습니다. 혼자 코딩하는 것을 넘어, 함께 코드를 만들고 서로의 아이디어를 섞어 더 나은 결과물을 만드는 과정이죠. 이를 위한 핵심 기능 세 가지가 바로 **Fork, Pull Request (PR), 그리고 충돌 해결**입니다.
>
> **Fork**는 다른 사람의 공개된 프로젝트 저장소를 내 GitHub 계정으로 그대로 복사해오는 기능입니다. 원본 프로젝트에는 전혀 영향을 주지 않고, 나만의 작업 공간에서 마음껏 코드를 수정하고 개선할 수 있죠. 그리고 이렇게 개선된 코드를 원본 프로젝트에 '이 부분을 이렇게 바꾸는 게 어때요?' 하고 제안하는 것이 바로 **Pull Request (PR)**입니다. PR을 통해 의견을 교환하고, 코드를 검토하며, 최종적으로 원본 프로젝트에 내 코드를 합치는 과정을 거칩니다. 만약 여러 사람이 같은 부분을 동시에 수정해서 충돌이 발생하면, GitHub이 친절하게 그 부분을 알려주고 어떻게 해결할지 가이드해 줍니다. 이러한 과정을 통해 학생들은 실제 개발 환경에서 필요한 협업 능력을 자연스럽게 기를 수 있습니다.

## [슬라이드 15] 현장 적용 아이디어 (1) – 학생 프로젝트 관리 & 학습 자료 버전 관리

**핵심 내용**
- 학생 개인 및 팀 코딩 프로젝트의 진행 과정, 기여도 투명하게 관리
- 수업 활동지, 코드 예시, 평가 문항 등 교사 학습 자료의 체계적인 버전 관리
- 협업 학습 및 프로젝트 기반 학습 (PBL) 효율성 증대

**강의 원고**
> 이제 Git과 GitHub을 우리 교실과 수업에 어떻게 적용할 수 있을지 구체적인 아이디어를 나눠보겠습니다. 첫째, **학생들의 코딩 프로젝트 관리**에 Git/GitHub을 활용해 보세요. 학생 개개인의 프로젝트든 팀 프로젝트든, GitHub 저장소를 만들어주면 학생들은 자신의 코드를 커밋하며 진행 과정을 기록할 수 있습니다. 누가 언제 어떤 코드를 작성했는지, 어떤 문제에 부딪혔고 어떻게 해결했는지 모든 이력을 투명하게 볼 수 있죠. 이를 통해 학생들의 학습 과정을 추적하고, 기여도를 평가하며, 적절한 피드백을 제공하는 데 큰 도움이 됩니다.
>
> 둘째, **교사 학습 자료의 버전 관리**입니다. 여러분이 만드는 수업 활동지, 코드 예시, 심지어는 파워포인트 자료나 평가 문항까지 Git 저장소로 관리해 보세요. 자료를 수정할 때마다 커밋을 남기면, 언제든 이전 버전으로 돌아갈 수 있고, 동료 교사들과 협업하여 자료를 개선할 때도 누가 어떤 부분을 수정했는지 명확하게 파악할 수 있습니다. 더 이상 '최종_진짜최종_수정본'이라는 파일 이름을 보며 혼란스러워할 필요가 없어집니다.

## [슬라이드 16] 현장 적용 아이디어 (2) – 교사 학습 공동체 및 교원 연수 자료 개발

**핵심 내용**
- 교사 학습 공동체(PLC)의 교육 콘텐츠 공동 개발에 Git/GitHub 활용
- 교원 연수 자료, 디지털 교과서 콘텐츠 등의 협력적 제작 및 공유
- 교사 간 지식 공유 및 전문성 신장 기회 확대

**강의 원고**
> 마지막으로, Git과 GitHub은 **교사 학습 공동체(PLC)** 활동에도 혁신적인 변화를 가져올 수 있습니다. 함께 연구하고 개발하는 교육 콘텐츠, 예를 들어 새로운 교과 과정에 맞는 수업 모듈, 학생들을 위한 웹 기반 학습 플랫폼, 또는 AI 활용 교육 연수 자료 등을 Git/GitHub 저장소에서 공동으로 개발해 보세요.
>
> 각자 맡은 부분을 브랜치로 나누어 독립적으로 작업하고, 완성되면 Pull Request를 통해 서로의 코드를 검토하고 피드백을 주고받는 과정을 거칠 수 있습니다. 이렇게 하면 누가 어떤 내용을 추가했는지, 어떤 피드백이 반영되었는지 모든 과정이 기록되어 책임감 있는 협업이 가능해집니다. '랜드마크 탐험관 V2'와 같은 토이 프로젝트를 개발하는 과정에서도 Git/GitHub을 활용하면 여러 교사가 함께 아이디어를 실현하고 발전시켜 나갈 수 있습니다. 교사 간의 지식 공유와 전문성 신장을 위한 강력한 플랫폼이 될 것입니다.

## [슬라이드 17] 마무리 – 오늘의 핵심 메시지 & 질문

**핵심 내용**
- **핵심 메시지 1: 실수는 성장의 기회입니다.** Git/GitHub은 안전한 시행착오를 가능하게 합니다.
- **핵심 메시지 2: AI를 넘어선 제어, 교사가 주도합니다.** AI의 도움을 받되, 프로젝트의 방향은 교사가 설정합니다.
- **핵심 메시지 3: 함께 만드는 교육, 협업의 가치를 높입니다.** Git/GitHub으로 효율적인 협업 환경을 구축합니다.

**강의 원고**
> 선생님들, 오늘 우리는 AI 시대 코딩 교육의 필수 도구인 Git과 GitHub에 대해 함께 알아보았습니다. 기억해야 할 세 가지 핵심 메시지를 다시 한번 강조하고 싶습니다.
>
> 첫째, **실수는 성장의 기회입니다.** Git과 GitHub은 여러분과 학생들이 코딩 과정에서 겪는 수많은 시행착오를 안전하게 관리하고, 언제든 이전 상태로 되돌릴 수 있는 든든한 안전망을 제공합니다. 더 이상 실수를 두려워하지 말고, 과감하게 도전해 보세요.
>
> 둘째, **AI를 넘어선 제어, 교사가 주도합니다.** AI는 강력한 조력자이지만, 프로젝트의 방향과 중요한 결정은 여전히 우리 교사와 학생들의 몫입니다. Git/GitHub을 통해 AI가 생성한 코드를 주도적으로 관리하고, 필요한 정보를 안전하게 보호하며, AI를 우리의 목적에 맞게 활용하는 능력을 키울 수 있습니다.
>
> 셋째, **함께 만드는 교육, 협업의 가치를 높입니다.** Git과 GitHub은 학생들의 협업 프로젝트를 효율적으로 만들고, 교사 학습 공동체 활동에서 교육 콘텐츠를 공동 개발하며, 서로의 지식을 공유하는 데 최적의 플랫폼입니다.
>
> Git과 GitHub은 단순한 기술을 넘어, AI 시대 교육의 불확실성 속에서 우리에게 안정감과 통제력을 선사하고, 협력적 성장을 가능하게 하는 강력한 도구입니다. 이제 교실에서 이 도구를 활용하여 새로운 교육의 문을 열어갈 여러분의 활약을 기대합니다.
