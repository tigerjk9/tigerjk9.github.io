## 핵심 작업 지침 (반드시 준수)

### 복수 출처 종합 — 번역이 아닌 재서술·통합

**절대 번역하지 않는다.** 각 아티클의 핵심 논지를 추출해 하나의 새로운 글로 재구성한다.

올바른 접근법:
1. 각 아티클을 읽고 **공통 주제**와 **각자의 독특한 논점**을 파악한다
2. 공통 주제를 중심으로 통합적 관점을 정립한다
3. 각 아티클의 독특한 시각·사례·수치를 활용해 논지를 풍부하게 만든다
4. 내가 이 주제를 오래 전부터 알고 있던 것처럼, **처음부터 새로 쓴다**

금지 사항:
- 각 아티클을 순서대로 나열하는 목차식 구성 ("첫 번째 글에서는…, 두 번째 글에서는…")
- 원문 문장을 한국어로 옮기는 것 (번역 금지)
- 출처를 명시하며 논지를 분리하는 것

허용 사항:
- 원문의 핵심 개념·사례·수치를 활용하되 표현 방식은 완전히 새롭게
- 한국 교육 현장의 구체적인 사례 추가
- 복수 아티클의 논리를 재배치·통합해 새로운 단일 논지 구성
- 필자 자신의 해석과 분석 추가

### 글쓰기 방식

독자는 원문을 못 읽은 한국의 교사·교육 관계자다.
이들에게 "내가 여러 흥미로운 글들을 읽었는데, 종합하면 이런 거야"라고 말해주는 방식으로 쓴다.

- 딱딱한 요약 나열이 아니라, 교육 현장 경험이 있는 선배가 들려주는 이야기처럼
- 복수 출처에서 뽑은 논점들이 하나의 흐름으로 자연스럽게 이어져야 한다
- 한국 독자가 바로 공감할 수 있는 맥락(한국 교육 현장, 학교 현실)을 중간중간 연결한다
- 여러 시각이 만나는 지점에서 필자만의 새로운 해석을 덧붙인다

### 본문

**분량**: 각 아티클의 핵심 논거를 빠짐없이 다룬다. 핵심 사례·수치·고유명사는 포함한다.

**구조**: `##` 헤딩으로 섹션을 나눈다. 복수 출처를 하나의 흐름으로 엮는다. 출처별 분리 구성 금지.

**서술 체크리스트** (각 섹션마다 하나 이상 반영):
- [ ] 복수 아티클의 논점이 하나의 흐름으로 자연스럽게 통합됐는가?
- [ ] 원문에 없는 나만의 비유나 예시를 추가했는가?
- [ ] 특정 아티클 기원이 드러나는 문장은 없는가?
- [ ] 필자의 해석이나 시각이 한 문장이라도 들어갔는가?

### 마지막 섹션: 다른 분야와의 연결

본문이 끝난 뒤, 딱 하나의 섹션을 추가한다.

섹션 제목은 반드시 아래 형식으로 쓴다:
```
## {CROSSOVER_DOMAIN}의 시선으로 보면
```

이 섹션의 규칙:
1. **{CROSSOVER_DOMAIN}** 분야의 개념 하나를 꺼내 본문 내용과 연결한다.
2. 억지 연결 금지 — 읽는 사람이 "아, 이건 진짜네" 싶어야 한다.
3. 길이는 3~5문장으로 짧게 유지한다.
4. **문체 다양성**: 매번 같은 구조를 반복하지 않는다.
   - 역설적 질문으로 시작: "왜 가장 안정적인 시스템이 가장 변하기 쉬운가?"
   - 단정적 선언으로 시작: "{CROSSOVER_DOMAIN}에는 이걸 설명하는 개념이 있다."
   - 대비로 시작: "A 분야에서는 이런 현상을 B라고 부른다. 교육에서 일어나는 일과 구조가 같다."
   - 구체적 장면으로 시작: 개념을 직접 보여주는 짧은 묘사
5. 금지: "~할 수 있다는 점이 흥미롭다", "~라는 점에서 유사하다"로 끝나는 결론

## 표 활용 규칙 (윤문화 방지 — 매우 중요)

표로 표현하면 더 명확한 내용을 서술 문장으로 풀어쓰지 않는다. 다음 신호가 보이면 즉시 표를 만든다.

**즉시 표로 변환 (예외 없음)**:
- 비교·대조 (A vs B, 전통 vs 신규, 이전 vs 이후 등)
- 분류·유형 (3가지 유형, 5가지 원칙, N단계 등 카운트가 명시된 항목)
- 항목별 속성 매트릭스 (각 항목이 동일한 속성·열을 가질 때)
- 수치·통계 비교 (효과 크기, 표준편차, 비율 등)
- 시간 흐름·연도별 변화
- 원칙·규칙 + 설명 + 효과·사례 같은 N열 구조

**표 작성 형식**:
- 표 위 한 줄로 도입 문장 (이 표가 무엇을 보여주는지)
- 마크다운 표준 표 (`| 헤더 | ... |`)
- 표 아래 한 줄 또는 두 줄로 핵심 해석·시사점만
- 표 내용을 다시 풀어 서술하지 않는다 (중복 설명 금지)

**서술로 남겨야 할 부분**:
- 도입·전환·종합 단락 (이야기 흐름)
- 사례·일화·인용 (특정 인물의 발언은 표에 가두지 않음)
- 추론·논증의 사고 흐름 (인과 관계 풀이)
- 한 줄 결론·핵심 정리

**금지**:
- "다음과 같다. 첫째, ~. 둘째, ~. 셋째, ~." 같은 첫째·둘째 윤문 나열 → 표로 변환
- 동일 속성을 가진 3개 이상 항목을 문장으로 풀어쓰기 → 표로 변환
- 표를 만든 뒤 "위 표에서 보듯이..."로 표 내용을 다시 서술

여러 출처의 공통점·차이점·관점을 비교할 때는 반드시 표로 정리한다 (출처 vs 출처 매트릭스).

## AI 티 금지 표현 (Humanize KR v2.0.0 — 발견 즉시 교체)

**S1 — 한 번만 나와도 교체**

| 금지 표현 | 대체 방법 |
|----------|---------|
| `~를 통해`, `~를 통한` | `~해서`, `~으로` |
| `~에 있어서`, `~에 있어` | `~에서` |
| `~을 넘어`, `~을 뛰어넘어` | 직접 단언 |
| `가지고 있다` | 구체 동사 (`갖는다`, `지닌다`) |
| `되어진다` (이중 피동) | `된다` |
| `결론적으로`, `요약하면`, `~라고 할 수 있다` | 삭제 |
| `시사하는 바가 크다`, `주목할 만하다`, `간과할 수 없다` | 구체 근거로 대체 |
| `매우 중요하다` | 왜 중요한지 논증으로 보여줌 |
| `크게 N가지로 나눌 수 있다`, `다음과 같은 특징이 있다` | 직접 나열 |
| `혁신적`, `획기적`, `전례 없는`, `압도적`, `막강한`, `폭발적`, `파격적`, `대대적`, `강력한`, `치열한`, `뜨거운` | 구체 수치·사례로 대체 |
| `~할 때입니다`, `~시점입니다` | 구체 동사 단언 |
| `'A'에서 'B'로` 변환 공식 | 직접 단언 |
| `## 제목: 부제` 콜론 헤딩 | 단일 명사구 또는 질문형 헤딩 |
| 이모지 (✅ 🚀 💡 등) | 전량 제거 |
| `채GPT` | `챗GPT` (ChatGPT 한국어 표기) |
| `제 해석으론` | `구조적으로 보면`, `본질적으로` 등으로 대체 |
| 한자 혼용(`陳述` 등), 괄호 내 영어 병기(`statement` 등) | 순우리말·한글 표기로 대체 |
| `그`, `그녀`, `그것`, `그들` 강박 직역 대명사 | 이름·역할·지시어(`이 도구`, `이 연구`)로 대체 |
| `~에서의`, `~에로의`, `~으로의`, `~에의` 이중 조사 | 자연스러운 단일 조사로 대체 |
| 연결어미 뒤 쉼표 (`하지만,` `그러나,` `따라서,`) | 쉼표 제거 |

**S2 — 3회 이상 반복 시 교체**

- `~할 수 있다` 종결 반복 → `~한다`로 단언
- `또한`, `따라서`, `즉` 문두 반복 → 70% 이상 삭제
- `이는 ~을 의미한다`, `이 관점에서` 반복 → 앞 문장과 합침
- `~것이다` 종결 반복 → `~다` 직결
- `~할 필요가 있다`, `~해야 한다` 5회 초과 → 주체·조건문으로 변주
- `점`, `바`, `수` 형식명사 반복 → 구체 명사·동사로 대체
- `첫째/둘째/셋째` 기계적 나열 → 일부 서술문으로 전환
- **볼드** 남용 → 핵심 개념·고유명사·수치만
- 관형구 3중 중첩 (`~가 만든 X 기반의 Y 설계` 등) → 문장 분리 또는 풀어쓰기
