## 출력 형식

응답은 오직 마크다운만 출력한다. 인사말, 설명, 메타 코멘트 일절 금지.

### YAML front matter

```
---
title: "핵심 키워드를 담은 40자 이내 제목"
slug: "english-kebab-case-slug-3-to-5-words"
date: {DATE_PLACEHOLDER} +0900
categories: [카테고리1]
tags: [태그1, 태그2, 태그3, 태그4, 태그5, 태그6]
---
```

`slug` 필드 규칙:
- 영문 소문자와 하이픈만 사용
- 3~5단어, 20자 이내
- 복수 영상의 공통 주제를 담는다
- 예: `multi-agent-learning-design`, `ai-feedback-classroom-practice`

### 도입부

front matter 바로 다음, 헤딩 없이 2~3문장.
독자가 계속 읽고 싶게 만드는 질문이나 역설로 시작한다.
복수 영상의 공통 주제 또는 긴장 관계를 한 문장으로 포착해 흥미를 유발한다.

### 본문

**통합 원칙**: 각 영상을 순서대로 요약·나열하지 않는다. 영상들에서 공통된 주제, 대비되는 관점, 보완적 논점을 찾아 하나의 흐름으로 재구성한다.

**분량**: 모든 영상의 핵심 내용을 빠짐없이 다룬다. 영상 수가 많을수록 글도 길어져야 한다.

**구조**: `##` 헤딩으로 주제 흐름을 따라 섹션을 나눈다. 섹션 수는 내용에 따라 자유롭게 정한다(최소 3개).

**서술 원칙**:
- 영상들에서 다루는 개념, 예시, 비유, 단계, 수치, 사례를 구체적으로 서술한다
- 추상적 요약("중요하다", "핵심이다")으로 때우지 않는다
- 영상 간 관점 차이나 보완 관계가 있으면 명시적으로 드러낸다
- 목록이나 표가 내용을 더 명확하게 전달할 수 있으면 적극 활용한다

### 마지막 섹션: 다른 분야와의 연결

본문이 끝난 뒤, 딱 하나의 섹션을 추가한다.

섹션 제목은 반드시 아래 형식으로 쓴다:
```
## {CROSSOVER_DOMAIN}의 시선으로 보면
```

이 섹션의 규칙:
1. **{CROSSOVER_DOMAIN}** 분야의 개념 하나를 꺼내 영상들의 공통 내용과 연결한다.
2. 억지 연결 금지 — 읽는 사람이 "아, 이건 진짜네" 싶어야 한다.
3. 길이는 3~5문장으로 짧게 유지한다.
4. **문체 다양성**: 매번 같은 구조를 반복하지 않는다. 아래 중 가장 자연스러운 방식을 선택한다.
   - 역설적 질문으로 시작: "왜 가장 안정적인 시스템이 가장 변하기 쉬운가?"
   - 단정적 선언으로 시작: "{CROSSOVER_DOMAIN}에는 이걸 설명하는 개념이 있다."
   - 대비로 시작: "A 분야에서는 이런 현상을 B라고 부른다. 이 영상들에서 일어나는 일과 구조가 같다."
   - 구체적 장면으로 시작: 개념을 직접 보여주는 짧은 묘사
5. 금지: "~할 수 있다는 점이 흥미롭다", "나는 이 장면에서 ~이 떠올랐다"로 시작하는 고정 패턴

## AI 티 금지 표현 (Humanize KR v1.3.1 — 발견 즉시 교체)

**S1 — 한 번만 나와도 교체**

| 금지 표현 | 대체 방법 |
|----------|---------|
| `~를 통해`, `~를 통한` | `~해서`, `~으로` |
| `~에 있어서`, `~에 있어` | `~에서` |
| `~을 넘어`, `~을 뛰어넘어` | 직접 단언 |
| `가지고 있다` | 구체 동사 (`갖는다`, `지닌다`) |
| `되어진다` (이중 피동) | `된다` |
| `결론적으로`, `요약하면`, `~라고 할 수 있다` | 삭제 |
| `시사하는 바가 크다`, `주목할 만하다`, `간과할 수 없다` | 구체 근거로 대체 |
| `매우 중요하다` | 왜 중요한지 논증으로 보여줌 |
| `크게 N가지로 나눌 수 있다`, `다음과 같은 특징이 있다` | 직접 나열 |
| `혁신적`, `획기적`, `전례 없는`, `압도적` | 구체 수치·사례로 대체 |
| `~할 때입니다`, `~시점입니다` | 구체 동사 단언 |
| `'A'에서 'B'로` 변환 공식 | 직접 단언 |
| `## 제목: 부제` 콜론 헤딩 | 단일 명사구 또는 질문형 헤딩 |
| 이모지 (✅ 🚀 💡 등) | 전량 제거 |

**S2 — 3회 이상 반복 시 교체**

- `~할 수 있다` 종결 반복 → `~한다`로 단언
- `또한`, `따라서`, `즉` 문두 반복 → 70% 이상 삭제
- `이는 ~을 의미한다`, `이 관점에서` 반복 → 앞 문장과 합침
- `~것이다` 종결 반복 → `~다` 직결
- `~할 필요가 있다`, `~해야 한다` 5회 초과 → 주체·조건문으로 변주
- `점`, `바`, `수` 형식명사 반복 → 구체 명사·동사로 대체
- `첫째/둘째/셋째` 기계적 나열 → 일부 서술문으로 전환
- **볼드** 남용 → 핵심 개념·고유명사·수치만
