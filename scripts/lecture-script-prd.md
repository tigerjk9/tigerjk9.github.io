# lecture-script PRD

> 설계 의도·구조·동작 원칙을 기록한 문서. `/yeonsu` 스킬의 유지보수 기준.

## 1. 목적

교사 연수를 자주 진행하는 사용자가 다양한 형태의 자료(YouTube 영상, 웹 아티클, PDF 논문, 텍스트 파일)를 제공하면, 그 내용을 분석해 **교원 연수 교재**를 자동 생성한다.

출력물은 두 가지 역할을 동시에 수행한다:
- **연수 교재** — 수강생이 읽고 학습하는 정식 교재. 기관 제출 가능.
- **블로그 포스트** — `_posts/`에 저장되어 웹에 발행 가능한 마크다운 문서

슬라이드 덱·강의 노트가 아닌, 호흡이 긴 에세이형 교재.

## 2. 출력 파일

| 파일 | 위치 | 용도 |
|------|------|------|
| `.md` | `_posts/` | 블로그 포스트 (Jekyll) |

## 3. 호출 방식

```
/yeonsu <입력> [옵션]
```

슬래시 커맨드가 강의 시간과 수준을 먼저 물어본 뒤 스크립트를 실행한다.

## 4. 입력 지원 범위

| 입력 타입 | 감지 조건 | 추출 방법 |
|-----------|-----------|-----------|
| YouTube URL | `youtube.com` 또는 `youtu.be` 포함 | youtube-transcript-api → yt-dlp VTT → description |
| 웹 URL | `http://` 또는 `https://`로 시작 | requests + BeautifulSoup → Jina Reader 폴백 |
| PDF 파일 | `.pdf` 확장자 | pdfplumber → PyMuPDF |
| 기타 파일 | 위 외 모든 경로 | python-docx → 텍스트 읽기 |

## 5. 파라미터

| 옵션 | 기본값 | 설명 |
|------|--------|------|
| `--duration N` | 120 | 강의 시간(분) |
| `--level` | 중급 | 강의 수준 (`초급` / `중급`) |
| `--date` | 오늘 | 포스트 날짜 |
| `--slug` | 자동 생성 | 파일명 슬러그 |
| `--no-push` | — | 로컬 저장만 |
| `--dry-run` | — | 출력만, 저장 없음 |
| `--model` | gemini-2.5-flash | Gemini 모델 ID |

## 6. 출력 구조

### Jekyll front matter
```yaml
---
title: "..."
date: YYYY-MM-DD 09:00:00 +0900
categories: [...]
tags: [...]
---
```

### 챕터 구조 (반복)
```markdown
## N. 챕터 제목

> *"에피그래프 인용"* — 출처

[케이스 오프너] 2~3문장. 구체적 상황 + 딜레마 질문으로 끝남. 특정 인물 주인공 금지.

[탐구 에세이 본문] 오프너 질문에서 출발해 개념·이론·연구를 사유의 흐름으로 전개. 최소 800자.

**토의 활동** *(전체 3~5개만)*
> "성찰 질문"

**핵심 정리**
> 챕터 핵심 메시지 한 문장
```

### 챕터 수 기준
- 챕터당 평균 45분 분량
- 60분 → 3~4챕터, 120분 → 3~4챕터, 360분 → 7~9챕터

### 참고문헌 (선택)
```markdown
## 참고문헌
- 저자명 (연도). 『서명』. 출판사.
```

## 7. HWPX 매핑

| 마크다운 요소 | HWPX 변환 |
|-------------|-----------|
| `## N. 챕터 제목` | `make_section_bar(N, 챕터 제목)` |
| 에피그래프 `> ` | `make_body_para('"', 내용)` |
| `**교실 이야기**` 아래 단락 | `make_body_para("◇", 내용)` |
| 일반 본문 텍스트 단락 | `make_body_para("▶", 내용)` |
| `**토의 활동**` `> ` | `make_body_para("Q.", 질문)` |
| `**핵심 정리**` `> ` | `make_body_para("■", 정리)` |
| `## 참고문헌` `- ` | `make_body_para("•", 참고문헌)` |

## 8. 관련 포스트 보강

`find_related_posts()` 로직:
1. 입력 콘텐츠 + 제목에서 한글 2자 이상, 영문 3자 이상 단어 추출
2. 불용어(`_STOP_WORDS`) 제거
3. `_posts/` 전체 순회 → 키워드 매칭 점수 산정
4. 상위 3개 포스트 본문(최대 1,500자) 프롬프트에 포함

## 9. 수준별 작성 원칙

**초급**
- 핵심 개념 2~3개에만 집중
- 심화 주제 제외
- 비유·실생활 예시 중심
- 전문 용어 필수 풀이

**중급**
- 전체 개념 다룸
- 심화 적용·비판적 시각 포함
- 연구 데이터 직접 인용

## 10. 알려진 동작 특성

- Gemini가 `date:` 연도를 임의로 바꾸는 버그 → 스크립트가 생성 후 강제 복원
- YouTube 자막 없는 영상 → description으로 자동 대체 (내용 빈약할 수 있음)

## 11. 파일 구조

```
scripts/
  lecture_script.py           # 메인 스크립트
  lecture_prompt_template.txt # Gemini 프롬프트 템플릿
  lecture-script-prd.md       # 이 문서
.claude/commands/
  yeonsu.md                   # /yeonsu 슬래시 커맨드
_posts/                       # 생성된 연수 교재 마크다운
_lectures/                    # 생성된 HWPX 파일
```
