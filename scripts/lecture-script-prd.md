# lecture-script PRD

> 설계 의도·구조·동작 원칙을 기록한 문서. `/yeonsu` 스킬의 유지보수 기준.

## 1. 목적

교사 연수를 자주 진행하는 사용자가 다양한 형태의 자료(YouTube 영상, 웹 아티클, PDF 논문, 텍스트 파일)를 제공하면, 그 내용을 분석해 **탐구 에세이 형식의 블로그 포스트**를 자동 생성한다.

출력물은 두 가지 역할을 동시에 수행한다:
- **현직 교사를 위한 탐구 글** — 교육 블로그에 발행되는 공개 포스트. "연수 교재", "수강생", "이 연수에서" 같은 표현은 절대 사용하지 않는다.
- **블로그 포스트** — `_posts/`에 저장되어 웹에 발행 가능한 마크다운 문서.

슬라이드 덱·강의 노트가 아닌, 호흡이 긴 **탐구 에세이** 형식.

## 2. 출력 파일

| 파일 | 위치 | 용도 |
|------|------|------|
| `.md` | `_posts/` | 블로그 포스트 (Jekyll) |

## 3. 호출 방식

```
/yeonsu <입력> [입력2 ...] [옵션]
```

입력값이 확인되면 **즉시 실행**. 시간·수준은 묻지 않고 기본값(120분/중급)으로 실행한다.
**복수 입력 지원**: 공백으로 구분해 여러 URL/파일을 전달하면 모두 추출·통합해 하나의 아티클로 생성.

## 4. 입력 지원 범위

| 입력 타입 | 감지 조건 | 추출 방법 |
|-----------|-----------|-----------|
| YouTube URL | `youtube.com` 또는 `youtu.be` 포함 | youtube-transcript-api → yt-dlp VTT → description |
| 웹 URL (일반) | `http://` 또는 `https://`로 시작 | requests + BeautifulSoup → Jina Reader 폴백 |
| 웹 URL (Naver 블로그) | `blog.naver.com` 포함 | `m.blog.naver.com`으로 자동 변환 후 모바일 UA로 추출 → Jina 폴백 |
| PDF 파일 | `.pdf` 확장자 | pdfplumber → PyMuPDF |
| 기타 파일 | 위 외 모든 경로 | python-docx → 텍스트 읽기 |

## 5. 파라미터

| 옵션 | 기본값 | 설명 |
|------|--------|------|
| `--duration N` | 120 | 강의 시간(분) |
| `--level` | 중급 | 강의 수준 (`초급` / `중급`) |
| `--date` | 오늘 | 포스트 날짜 |
| `--slug` | 자동 생성 | 파일명 슬러그 |
| `--no-push` | — | 로컬 저장만 |
| `--dry-run` | — | 출력만, 저장 없음 |
| `--model` | gemini-2.5-flash | Gemini 모델 ID |

## 6. 출력 구조

### Jekyll front matter
```yaml
---
title: "핵심 메시지가 담긴 40자 이내 제목"
date: YYYY-MM-DD 09:00:00 +0900
categories: [...]
tags: [...]
---
```

### 도입부
front matter 직후, 헤딩 없이 2~3문장. 이 글에서 무엇을 탐구하는가.

### 본문 챕터 구조 (반복)
```markdown
## N. 챕터 제목 (질문형 또는 핵심 주장형)

> *"에피그래프"*

케이스 오프너: 교육 현장의 구체적 상황 2~3문장. 마지막 문장은 딜레마 질문.
특정 인물(이름/직함)을 주인공으로 내세우지 않는다.

탐구 에세이 본문: 오프너 질문에서 출발해 개념·이론·연구를 사유의 흐름으로 전개. 최소 800자.

**토의 활동** *(전체 문서에서 3~5개만. 모든 챕터에 달지 않는다)*
> "파트너와 5분간 이야기 나눌 수 있는 성찰 질문."

**핵심 정리**
> 이 챕터에서 가장 중요한 메시지를 한 문장으로 요약.
```

### 마지막 챕터 구조 (갈무리)
```markdown
## N. 챕터 제목 (전체 글의 핵심 질문에 대한 답 또는 열린 결말형)

앞선 챕터들의 핵심 논점을 연결해 종합하는 본문. 최소 400자.

**앞으로**
교실·학교·자기 자신에게 이어갈 방향. 처방이 아니라 가능성으로 서술.

**생각할 질문**

> "열린 질문 1 — 정답이 없는 질문."

> "열린 질문 2 — 독자가 자신의 현장에 대입해 볼 수 있는 질문."

> "열린 질문 3 — 이 글의 탐구를 더 멀리 밀고 나가는 질문."
```

### 챕터 수 기준
- 챕터당 평균 45분 분량
- 60분 → 3~4챕터, 120분 → 3~4챕터, 360분 → 7~9챕터

### 참고문헌 (선택)
```markdown
## 참고문헌
- 저자명 (연도). 『서명』. 출판사.
```

## 7. 에피그래프 규칙

- **확신 기준**: 저서명·출판연도를 즉시 댈 수 있을 정도로 널리 알려진 인용 → `— 이름` 출처 표기
- **불확실**: 출처 없이 경구 형식만 `> *"문장."*`
- **절대 금지**: 한국인 학자 이름 날조, "교육공학자 ○○○" 형식의 직함+이름 조합, 실존 여부 불분명한 인물 출처 표기

## 8. 관련 포스트 보강

`find_related_posts()` 로직:
1. 입력 콘텐츠 + 제목에서 한글 2자 이상, 영문 3자 이상 단어 추출
2. 불용어(`_STOP_WORDS`) 제거
3. `_posts/` 전체 순회 → 키워드 매칭 점수 산정
4. 상위 3개 포스트 본문(최대 1,500자) 프롬프트에 포함

## 9. 수준별 작성 원칙

**초급**
- 핵심 개념 2~3개에만 집중
- 심화 주제 제외
- 비유·실생활 예시 중심
- 전문 용어 필수 풀이

**중급**
- 전체 개념 다룸
- 심화 적용·비판적 시각 포함
- 연구 데이터 직접 인용

## 10. 알려진 동작 특성

- Gemini가 `date:` 연도를 임의로 바꾸는 버그 → 스크립트가 생성 후 강제 복원
- YouTube 자막 없는 영상 → description으로 자동 대체 (내용 빈약할 수 있음)
- Gemini가 전체 출력을 ` ```markdown ``` `으로 감싸는 경우 → 스크립트가 후처리로 제거
- **Naver 블로그**: `blog.naver.com` → `m.blog.naver.com` 자동 변환 + 모바일 UA로 로그인 없이 본문 추출 가능. `browser_cookie3`(Chrome/Edge) 또는 `NAVER_COOKIE` 환경변수로 쿠키를 주입하면 회원 전용 콘텐츠도 접근 가능하나, Windows에서 `browser_cookie3`는 관리자 권한 필요.

## 11. 문체 규칙

- 어미: `~이다`, `~한다`, `~된다` 단정체
- 금지: `~입니다`, `~합니다`, `~됩니다`, `~함`, `~됨`
- 금지 표현: "선생님들 안녕하세요", "함께 살펴보자", "흥미롭게도", "주목할 필요가 있다"
- 인물 드라마 금지: 특정 인물을 주인공으로 한 3인칭 묘사 전면 금지

## 12. 파일 구조

```
scripts/

.claude/commands/
  yeonsu.md                   # /yeonsu 슬래시 커맨드
_posts/                       # 생성된 포스트 마크다운
```
