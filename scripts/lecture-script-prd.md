# lecture-script PRD

> 설계 의도·구조·동작 원칙을 기록한 문서. `/lecture-script` 스킬의 유지보수 기준.

## 1. 목적

교사 연수를 자주 진행하는 사용자가 다양한 형태의 자료(YouTube 영상, 웹 아티클, PDF 논문, 텍스트 파일)를 제공하면, 그 내용을 분석해 **교사 연수용 강의 스크립트**를 자동 생성한다.

출력물은 두 가지 역할을 동시에 수행한다:
- **강의 원고** — 발표자가 실제로 읽는 구어체 대본
- **블로그 포스트** — `_posts/`에 저장되어 웹에 발행 가능한 마크다운 문서

## 2. 출력 파일

| 파일 | 위치 | 용도 |
|------|------|------|
| `.md` | `_posts/` | 블로그 포스트 (Jekyll) |
| `.hwpx` | `_lectures/` | 강의 원고 문서 (한컴오피스) |

HWPX는 `hwpx` 전역 스킬 Workflow A (government 템플릿)로 생성.
슬라이드 섹션 → section bar, 강의 원고 → `▶` 마커, 발문 → `Q.` 마커로 매핑.

## 3. 호출 방식

```
/lecture-script <입력> [옵션]
```

슬래시 커맨드가 강의 시간과 수준을 먼저 물어본 뒤 스크립트를 실행한다.

## 3. 입력 지원 범위

| 입력 타입 | 감지 조건 | 추출 방법 |
|-----------|-----------|-----------|
| YouTube URL | `youtube.com` 또는 `youtu.be` 포함 | youtube-transcript-api → yt-dlp VTT → description |
| 웹 URL | `http://` 또는 `https://`로 시작 | requests + BeautifulSoup → Jina Reader 폴백 |
| PDF 파일 | `.pdf` 확장자 | pdfplumber → PyMuPDF |
| 기타 파일 | 위 외 모든 경로 | python-docx → 텍스트 읽기 |

## 4. 파라미터

| 옵션 | 기본값 | 설명 |
|------|--------|------|
| `--duration N` | 120 | 강의 시간(분) |
| `--level` | 중급 | 강의 수준 (`초급` / `중급`) |
| `--date` | 오늘 | 포스트 날짜 |
| `--slug` | 자동 생성 | 파일명 슬러그 |
| `--no-push` | — | 로컬 저장만 |
| `--dry-run` | — | 출력만, 저장 없음 |
| `--model` | gemini-2.5-flash | Gemini 모델 ID |

## 5. 출력 구조

### Jekyll front matter
```yaml
---
title: "..."
date: YYYY-MM-DD 09:00:00 +0900
categories: [...]
tags: [...]
---
```

### 강의 개요 표
| 항목 | 내용 |
|------|------|
| 대상 | 교원 연수 |
| 시간 | N분 |
| 수준 | 초급/중급 |
| 핵심 목표 | ... |

### 슬라이드 섹션 (반복)
```markdown
## [슬라이드 N] 제목

**핵심 내용**
- 포인트 1
- 포인트 2

**강의 원고**
> 구어체 대본. 분당 250~300자 기준.

**발문** *(전체 3~5개만)*
> "열린 질문"
```

### 슬라이드 수 기준
- 슬라이드당 평균 4분 산정 (원고 낭독 + 발문 + 전환)
- 120분 → 28~34개 슬라이드 범위로 계산

## 6. 관련 포스트 보강

`find_related_posts()` 로직:
1. 입력 콘텐츠 + 제목에서 한글 2자 이상, 영문 3자 이상 단어 추출
2. 불용어(`_STOP_WORDS`) 제거 — 교육·학교·교사 등 변별력 없는 단어 제외
3. `_posts/` 전체 순회 → 각 포스트의 제목·태그·카테고리에서 키워드 매칭 점수 산정
4. 상위 3개 포스트 본문(최대 1,500자) 프롬프트에 포함
5. Gemini 지침: "복붙 금지, 강의 흐름에 자연스럽게 녹여 넣기"

## 7. 수준별 작성 원칙

**초급**
- 전문 용어를 쉬운 말로 풀어 설명
- 비유·실생활 예시 중심
- 이론보다 실천 방법에 비중
- 발문은 경험을 떠올리는 쉬운 질문

**중급**
- 기본 개념은 알고 있다고 가정
- 심화 적용과 비판적 시각 다룸
- 연구 결과·데이터 직접 인용
- 발문은 실천 성찰을 유도하는 질문

## 8. 알려진 동작 특성

- Gemini가 `date:` 연도를 임의로 바꾸는 버그 → 스크립트가 생성 후 강제 복원
- 한국어 제목 → 슬러그 직접 추출 불가 → Gemini slug 생성으로 해결
- 기업 네트워크 SSL 오류 → `ssl._create_unverified_context` + requests 세션 패치로 우회
- YouTube 자막 없는 영상 → description으로 자동 대체 (내용 빈약할 수 있음)
