## 작업 원칙 — 반드시 준수

### 1. 기존 구조 우선
- 기존 섹션 제목·순서는 가능한 한 그대로 보존한다
- 새 섹션 추가는 신규 자료가 **독립된 관점·층위**를 제공할 때에 한해 허용한다 (예: 비판적 균형, 새로운 분류 체계, 별도의 메커니즘 분석)
- 도입부와 마무리는 기존 톤을 유지한 채 강화하는 정도로 다룬다

### 2. 신규 자료에서 채굴할 것
다음을 우선순위로 발굴해 기존 본문에 녹인다:
- **충격적 수치·통계**: 도입부 또는 가장 가까운 섹션에서 근거를 강화한다
- **저자의 인상적 비유·은유·표현**: 기존에 추상적으로 서술된 부분을 구체화한다
- **잘 알려지지 않은 인용·출처**: 개념 제안자 이름, 학자 출처, 인용된 학자의 핵심 문장을 명시한다
- **구체적 사례·실험 결과·수치**: 일반론을 받쳐주는 증거로 끼워 넣는다
- **기존에 없는 균형 관점**: 비판·한계·반론 — 이 경우 별도의 새 섹션 신설이 정당하다
- **구조적·정책적 대안**: 해결책 섹션이 있다면 더 구체적인 대안으로 보강한다

### 3. 통합 방식
- 신규 자료의 표현을 그대로 옮기지 않는다 (번역 금지)
- 기존 문장에 한두 절을 덧붙이거나, 기존 단락 사이에 한 단락을 끼워 넣는 식으로 자연스럽게 흡수한다
- 같은 사실을 두 번 말하지 않는다 — 기존 본문에 이미 있는 내용은 강화만 하고 중복 추가는 금지한다
- 신규 자료의 비판적 관점이 기존 글과 상충할 경우, 양쪽을 균형 있게 서술하되 사용자의 기존 관점을 임의로 뒤집지 않는다

### 4. 메타데이터
- **title**: 기존 제목 유지를 원칙으로 한다 (변경할 만한 명확한 이유가 없는 한 그대로)
- **date**: 기존 값을 그대로 유지한다 — **절대 바꾸지 않는다**. 시·분·초까지 한 글자도 손대지 않는다
- **categories**: 기존 그대로 유지한다
- **tags**: 신규 자료를 반영하는 1~2개 태그만 끝에 추가한다. 기존 태그는 모두 유지한다
- **<출처>**: 기존 항목을 모두 유지한 채, 신규 URL을 끝에 한 줄씩 추가한다

### 5. 문체·표기 (기존 포스트와 완전 일치)
- 단정체 (`~이다`, `~한다`, `~된다`, `~았다`)
- 금지 어미: `~입니다`, `~합니다`, `~됩니다`, `~함`, `~됨`
- 핵심 개념·고유명사는 **볼드** 처리
- **따옴표 사용 금지**: 큰따옴표 ", 작은따옴표 ' 모두 금지한다. 강조는 볼드로, 인용은 따옴표 없이 자연스러운 화법으로 변환한다
- AI 슬롭 금지: "함께 살펴보자", "주목할 필요가 있다", "중요한 시사점", "흥미롭게도", "놀랍게도"

### 6. 크로스오버 섹션
- 기존 포스트에 크로스오버 섹션(예: `## XX의 시선으로 보면`)이 있으면 **그대로 유지**한다
- 분야명을 **변경하지 않는다** — 이미 정해진 분야의 시선을 그대로 따른다
- 신규 통합 내용과 자연스럽게 연결되는 한두 문장을 보강할 수는 있다
- 기존에 크로스오버 섹션이 없으면 새로 추가하지 않는다

### 7. 신규 섹션 추가 가이드
새 섹션 신설은 다음 중 하나에 해당할 때만 한다:
- 신규 자료에 책·자료의 **한계나 비판적 관점**이 있어 균형감이 필요할 때
- 신규 자료가 기존 분류 체계를 **확장하는 새 층위**를 제공할 때 (예: 기존 3분류에 4번째 항목 추가)
- 신규 자료에 기존 섹션 어디에도 자연스럽게 들어가지 못하는 **독자적 관점**이 있을 때

신설 시 위치는 본문의 흐름을 깨지 않는 곳을 택한다. 보통 마지막 섹션 직전 또는 결론적 섹션 앞이 자연스럽다.
