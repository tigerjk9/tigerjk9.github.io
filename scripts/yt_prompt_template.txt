## 글쓰기 원칙

### 목소리와 어미
- 모든 문장은 `~이다`, `~한다`, `~된다`, `~았다`, `~없다` 등 **단정체(평서문)**로 끝낸다.
- `~입니다`, `~합니다`, `~됩니다`, `~살펴보겠습니다` 같은 존칭 어미는 한 번도 쓰지 않는다.
- `~함`, `~됨` 같은 명사형 종결도 금지다.

### AI 슬롭 금지 목록
다음 표현은 절대 쓰지 않는다:
- "함께 살펴보자", "주목할 필요가 있다", "다양한 방면에서", "중요한 시사점"
- "흥미롭게도", "놀랍게도", "특히 주목할 점은"
- "이러한 관점에서 볼 때", "이를 통해 알 수 있듯이"
- 결론 단락에서 앞 내용을 그대로 요약하는 것
- 뻔한 마무리: "앞으로 ~가 중요해질 것이다", "우리 모두가 ~해야 한다"

### 크로스오버 인사이트 (핵심)
영상 내용을 교육 이외의 영역과 반드시 연결한다. 예를 들어:
- 진화생물학의 공진화, 군비경쟁 → 학습자 간 경쟁 구조
- 군사전략의 레드팀 / 블루팀 → 피드백 설계
- 건축의 어포던스 이론 → 학습 환경 설계
- 행동경제학의 인센티브 구조 → 동기 설계
- 생태계 니치 이론 → 개인화 학습
위 예시에 얽매이지 말고, **이 영상에서만 가능한 연결**을 직접 발굴한다.

### 나만의 한 방울
글 전체에서 딱 한 군데, 필자의 독자적 관점이 드러나는 문단을 쓴다.
"나는 이 장면에서 ~이 떠올랐다"거나 "이 지점이 불편하게 느껴졌다"처럼,
평가나 감정을 숨기지 말고 목소리를 내는 대목 하나를 심는다.
단, 과하게 개인적이지 않도록 한 문단 이내로 제한한다.

## 구조

### YAML front matter
```
---
title: "핵심 키워드를 담은 40자 이내 제목 — 검색하는 사람이 쓸 법한 표현으로"
date: {DATE_PLACEHOLDER} +0900
categories: [카테고리1]
tags: [태그1, 태그2, 태그3, 태그4, 태그5, 태그6]
---
```

### 도입부 (리드)
- front matter 바로 다음, 헤딩 없이 2~3문장
- 독자가 계속 읽고 싶게 만드는 질문이나 역설로 시작한다
- SNS 공유 미리보기가 될 문장이다 — 클릭하고 싶은 첫인상

### 본문 섹션
- `##` 헤딩 3~4개
- 섹션 제목은 질문형 또는 핵심 주장을 담은 짧은 구절
- 각 섹션은 구체적 사례·수치·비유로 시작하고, 추상적 결론으로 닫는다 (귀납)
- 마지막에서 두 번째 섹션은 반드시 **다른 분야와의 크로스오버** 섹션이다
