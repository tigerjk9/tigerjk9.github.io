**앞으로**

구체적인 방향이나 실천의 씨앗을 제시한다. 처방이 아니라 가능성으로 서술한다.

### 에피그래프 규칙 (엄수)

- 해당 인물의 실제 저작·발언임을 확신할 수 있는 경우에만 `— 이름` 출처를 표기한다.
- **확신 기준**: 저서명·출판연도를 즉시 댈 수 있을 정도로 널리 알려진 인용. 예: 존 듀이 『경험과 교육』, 비고츠키의 ZPD 개념 등.
- 확신할 수 없으면 출처 없이 경구 형식으로만 쓴다: `> *"문장."*`
- **절대 금지**: 한국인 학자 이름 날조, "교육공학자 ○○○" 형식의 직함+이름 조합, 실존 여부가 불분명한 인물 출처 표기. 의심스러우면 무조건 출처 생략.
- 출처를 생략해도 에피그래프 자체는 쓸 수 있다. 강렬한 문장이 중요하다.

### 케이스 오프너 작성 원칙

- 2~3문장. 길면 안 된다.
- 특정 인물(이름, 직함)을 주인공으로 내세우지 않는다.
- 드라마 묘사 금지: "김 교사는 고민이 많다", "수진이는 눈을 감았다" 같은 문장 금지.
- 반드시 딜레마 질문으로 끝난다. 독자가 스스로 답을 찾고 싶게 만들어야 한다.

### 탐구 에세이 본문 서술 원칙

- 오프너의 질문을 챕터 전체의 탐구 축으로 삼는다.
- 개념을 불릿으로 나열하지 않는다. 서술문으로 풀어 설명한다.
- "~이 중요하다", "주목할 필요가 있다" 같은 단정 선언 금지. 왜 중요한지를 논증으로 보여준다.
- 이론과 현장의 긴장을 유지한다.
- 수치·연구 결과·고유명사는 원자료에서 정확하게 인용한다.
- 각 챕터 본문은 최소 800자 이상.

### 챕터 흐름 설계

- **챕터 1**: 도입 — 이 글의 핵심 질문 제시. 왜 지금 이 주제인가.
- **챕터 2~(N-1)**: 핵심 주제 — 원자료의 주요 논점을 교사 관점으로 재구성.
- **챕터 N**: 갈무리 — 인사이트 통합, 방향 제시, 열린 질문 3개.

## 문체 (전 문서 공통)

## AI 티 금지 표현 (Humanize KR v1.3.1 — 발견 즉시 교체)

**S1 — 한 번만 나와도 교체**

| 금지 표현 | 대체 방법 |
|----------|---------|
| `~를 통해`, `~를 통한` | `~해서`, `~으로` |
| `~에 있어서`, `~에 있어` | `~에서` |
| `~을 넘어`, `~을 뛰어넘어` | 직접 단언 |
| `가지고 있다` | 구체 동사 (`갖는다`, `지닌다`) |
| `되어진다` (이중 피동) | `된다` |
| `결론적으로`, `요약하면`, `~라고 할 수 있다` | 삭제 |
| `시사하는 바가 크다`, `주목할 만하다`, `간과할 수 없다` | 구체 근거로 대체 |
| `매우 중요하다` | 왜 중요한지 논증으로 보여줌 |
| `크게 N가지로 나눌 수 있다`, `다음과 같은 특징이 있다` | 직접 나열 |
| `혁신적`, `획기적`, `전례 없는`, `압도적` | 구체 수치·사례로 대체 |
| `~할 때입니다`, `~시점입니다` | 구체 동사 단언 |
| `'A'에서 'B'로` 변환 공식 | 직접 단언 |
| `## 제목: 부제` 콜론 헤딩 | 단일 명사구 또는 질문형 헤딩 |
| 이모지 (✅ 🚀 💡 등) | 전량 제거 |

**S2 — 3회 이상 반복 시 교체**

- `~할 수 있다` 종결 반복 → `~한다`로 단언
- `또한`, `따라서`, `즉` 문두 반복 → 70% 이상 삭제
- `이는 ~을 의미한다`, `이 관점에서` 반복 → 앞 문장과 합침
- `~것이다` 종결 반복 → `~다` 직결
- `~할 필요가 있다`, `~해야 한다` 5회 초과 → 주체·조건문으로 변주
- `점`, `바`, `수` 형식명사 반복 → 구체 명사·동사로 대체
- `첫째/둘째/셋째` 기계적 나열 → 일부 서술문으로 전환
- **볼드** 남용 → 핵심 개념·고유명사·수치만
