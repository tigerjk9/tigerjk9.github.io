오늘 가져갈 산출물 세 개

  1. 본인 폴더 CLAUDE.md 한 개 (강의 톤 · 출력 규칙 5줄)
  2. .claude/settings.json의 deny 한 줄
  3. 호출 가능한 /<스킬-이름> 한 개 (SKILL.md 5줄)

덤. Advanced에서 한 번 더 — 본인 파이프라인에 끼울 후보 1~2개 머리에 담기.

오늘의 약속 세 가지

  1. 모든 명령은 PowerShell 7+. 한글 폴더 경로 피하고 ASCII (예. C:\work\demo)
  2. 막히면 손. 한 사람의 막힘은 다섯 사람의 막힘
  3. 22개 다 외우려 하지 않기. 와닿는 1~2개만 가져가기

한글이 깨질 때. PowerShell에 chcp 65001 한 줄 — 가장 먼저 입력할 응급 처방.

B1 CLAUDE.md + @import
Basic 1
프로젝트 루트의 마크다운 한 파일. 매 세션 자동 로드. 4계층(managed/user/project/local)이 위계로 겹쳐 적용.
PowerShell
PS C:\work\my-project> claude # 현재 폴더에서 시작 > /init # CLAUDE.md 초안 자동 생성 > notepad CLAUDE.md # 메모장으로 5~10줄 다듬기 (@import로 공용 규칙 끌어쓰기)
Prompt이 폴더에 CLAUDE.md를 만들어 줘. 답변은 한국어로, 학생 원본 답안은 절대 수정하지 말고, 본인 강의의 표기 규칙을 따르도록.
한 폴더에 하나만 두면 모든 수업 자료의 톤이 한결같아집니다.
B2 Auto Memory
Basic 2
정정한 걸 자동 저장. 내 디스크에 마크다운으로 누적. ChatGPT 메모리와 달리 직접 열어 보고 지울 수 있어요.
Slash · env
> /memory # Auto Memory 토글 + MEMORY.md 폴더 열기 > /memory off # 이번 세션만 자동 누적 멈춤 (민감 정정 직전) PS> $env:CLAUDE_CODE_DISABLE_AUTO_MEMORY=1 # 환경변수로 영구 비활성
Prompt앞으로 표는 항상 마크다운 표로 출력해 줘. 우리 강의팀 표기 규칙이야. 다음부터 그렇게 가 줘.
민감 정정(학생 개인정보·미공개 평가 기준)은 off로. 수업 준비 ON / 학생 공유 OFF 분리 권장.
B3 Plan Mode
Basic 3
읽기만 + 변경 계획만. 승인 후 실행. AI가 멋대로 고치는 우려에 직접 답하는 모드.
CLI · Slash
PS> claude --permission-mode plan # Plan 모드로 시작 (실행 없이 계획만) > /plan # 세션 안에서 Plan 모드로 전환 키보드. Shift+Tab Shift+Tab # 모드 토글 (default → acceptEdits → plan)
Prompt이 학습지 12페이지를 8페이지로 줄여 줘. 어디를 어떻게 줄일지 먼저 계획만 보여 줘. 실행은 아직 하지 마.
수업 자료 다듬기 전 Plan으로 구조부터 받기 — 본문 수정량 절반. 함정. 읽기 명령은 실행됩니다 — Get-ChildItem·Get-Content는 동작.
B4 Permission Modes 6종
Basic 4
자율성 다이얼 — plan · default · acceptEdits · auto · dontAsk · bypass. 일상은 default · acceptEdits 둘만.
settings.json
// .claude/settings.json { "defaultMode":"acceptEdits", // 기본 모드 (변경은 자동, 터미널은 묻기) "permissions":{ "deny":["Bash(Remove-Item *)"], // deny = 항상 거부 (위험 명령) "allow":["Bash(git status)"] // allow = 항상 허용 (안전·반복 명령) } }
Prompt내 .claude/settings.json을 만들어 줘. deny에 Bash(Remove-Item *)·Read(~/.ssh/**), ask에 Bash(git push *), allow에 Bash(git status)·Bash(Rscript *)·Read(./data/**). defaultMode는 acceptEdits.
bypass는 격리 환경(컨테이너·VM·일회용 폴더) 외 절대 금지.
B5 Skills (/스킬-이름)
Basic 5
절차를 SKILL.md 한 파일로. 매일 한 줄 호출. description만 평상시 색인 — 토큰 0 / 호출 시만 로드.
PowerShell · Slash
PS> mkdir ~/.claude/skills/pdf-요약 # 스킬 폴더 (이름 = 호출 명령어) PS> notepad ~/.claude/skills/pdf-요약/SKILL.md # 절차·도구·경로 정의 > /pdf-요약 ./이 문서.pdf # 슬래시 명령으로 호출 + 인자 전달 # 옵션. paths: ["**/*.pdf"] (특정 파일만 자동) · context: fork (서브에이전트 격리)
Prompt내 ~/.claude/skills/ 아래에 본인이 자주 하는 작업 이름으로 스킬을 만들어 줘. PDF 한 편을 받아 (1) 한 문장 요약 (2) 5단락 한국어 요약 (3) 핵심 인용 5개(페이지 포함) (4) 시사점 3개. allowed-tools=Read,Grep · paths=**/*.pdf · shell=powershell.
자주 하는 작업을 스킬로 만들면 한 줄 호출로 끝나 매번 입력하던 시간이 줄어듭니다.
B6 Subagents
Basic 6
별도 컨텍스트에서 작업하고 요약만 메인에 돌려주는 전문 동료. 역할의 분리(스킬은 절차 재사용).
PowerShell · @mention
PS> mkdir .claude\agents # 프로젝트 전용 에이전트 폴더 PS> notepad .claude\agents\deep-reviewer.md # 에이전트 정의 (frontmatter + 본문) > @deep-reviewer 평가지·수업안 초안을 검토해 줘 # @이름으로 서브에이전트 직접 호출 (연구자면 논문 초고)
Prompt.claude/agents/ 아래에 'deep-reviewer'를 정의해 줘. 역할은 까다로운 출제위원 시각으로 문항 적절성·난이도·정답 근거·논리를 본다. tools=Read,Grep,Glob · model=opus · isolation=worktree. initialPrompt에 "이 폴더의 .md 초고를 찾아 검토 시작".
tools 화이트리스트 + isolation: worktree 두 줄 잊지 말기.

B7 /agents Library/Running
Basic 7
정의된 에이전트 목록(Library)과 라이브 인스턴스(Running)를 한 화면에서. 동료 에이전트 관제 화면.
Slash · Keys
> /agents # 에이전트 패널 열기 (Library = 정의 / Running = 진행 중) # Library ↔ Running Tab 키 # 두 탭을 키보드로 전환 # 같은 에이전트 다중 인스턴스 가능 # 출제위원-reviewer 다섯 명을 동시에 띄움
Prompt출제위원-reviewer 서브에이전트한테 이 폴더의 평가지·수업안 초안을 검토하라고 시켜 줘. 끝나면 결과만 메인 세션에 요약해 줘.
5명 동료 교육자 검토를 한 화면에. Esc로 특정 인스턴스만 멈춤.
B8 /compact · /clear
Basic 8
컨텍스트 관리. /compact는 같은 일 이어가기(자동 재주입), /clear는 새 일 시작. 자동 임계 + 수동.
Slash
> /compact # 지금까지 대화를 요약·압축 (같은 일을 계속 이어갈 때) > /clear # 컨텍스트 완전 초기화 (전혀 다른 일로 넘어갈 때) > /context # 현재 사용량 먼저 확인 (compact 결정 전 진단)
Prompt지금까지 분석 흐름을 5문장으로 요약하고 컨텍스트를 압축해 줘. 표·코드·이미지 경로는 그대로 남겨 두고.
자동 compact 직전에 핵심 보존 한 줄을 남겨 두기.
B9 /usage
Basic 9
토큰·비용·세션 통계. 영수증 역할. 단체 라이선스 환경에서 본인 페이스 잡는 용도.
Slash
> /usage # 이번 주 토큰·비용 요약 한 화면 (한도 대비 잔량) > /extra-usage # 세션·시간대별 상세 분석 (어디서 많이 썼는지 추적)
Prompt비용 많이 나간 세션 톱3이랑 그 이유를 같이 짚어 줘.
"학습 자료 8개 합치고 4달러" 같은 구체 감각.
B10 --continue · --resume
Basic 10
직전 세션 또는 이름·ID로 임의 세션 부활. --fork-session으로 분기. 회의·실험으로 끊겨도 한 줄로.
CLI · Keys
PS> claude -n 2026-1st-semester # 새 세션에 이름 부여 (-n = name) PS> claude --continue # 직전 세션 맥락을 그대로 이어 시작 PS> claude --resume 2026-1st-semester # 이름·ID로 임의 세션 부활 (며칠 전 것도 OK) PS> claude --fork-session # 같은 출발점에서 새 세션으로 분기 (다른 시도) Ctrl+A # picker 안에서 모든 프로젝트 세션 펼치기
Prompt오전에 시작한 학기 보고서 작성 세션 이어서 가 줘.
오전 수업 준비 → 오후 평가표 정리 재개. 끊긴 맥락을 한 줄로.
B11 Agent Teams (실험)
Basic 11
한 lead가 N teammates를 띄우고 공유 task list로 협업. 실험 기능 — 환경변수로 켜고 끔.
settings.json
// settings.json { "env":{ "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS":"1" } } // 실험 기능 플래그 (팀 모드 활성) > /agents → spawn // 활성 후 lead가 자동 분배·spawn
Prompt문항 적절성·정답 근거·표현(가독성) 동료 교육자 세 명이 동시에 검토하도록 팀 짜 줘. lead는 통합 보고만 맡고.
Basic에서는 "있다는 것만". 토큰 소비 큼 — /usage로 모니터링.
B12 .claude/rules/ + paths
Basic 12
CLAUDE.md를 토픽별로 쪼개고 글롭 매칭 시점에만 로드. 토큰 0 / 정확도 1 — raw 데이터 만질 때만 떠 줌.
rules.md frontmatter
// .claude/rules/data-files.md --- paths: ["data/원본/**"] // 글롭 매칭될 때만 자동 로드 (토큰 0 / 정확도 1) --- - 원본 폴더 파일은 수정 금지 // 룰 본문 = 매칭 시 추가되는 시스템 규칙 - 정제 결과는 _clean 접미사로
Prompt이 .claude/rules/data-files.md 좀 짜 줘. 원본 자료(학생 명단·답안·실험 데이터) 폴더의 파일은 절대 수정 금지, 새 파일은 _clean 접미사로.
팀원이 실수로 원본을 건드릴 가능성 차단. 데이터 무결성의 핵심.

A1 헤드리스 (-p + JSON)
Adv 1
claude -p 비대화형. stdin 파이프 입력 + JSON 구조화 출력. 셸 한 줄로 AI를 파이프라인에 끼우기.
PowerShell pipeline
PS> Get-Content students.csv | claude -p ` # -p = 헤드리스 모드 (대화창 없이 stdin 입력) "표를 학기 보고서 문체로" ` # 변환 지시 (한 줄 프롬프트) --output-format json --max-budget-usd 1.00 ` # JSON 출력 + 비용 상한 (넘으면 자동 중단) | Set-Content out.json # 결과를 새 파일로 저장 # 옵션. --json-schema '{...}' # 응답을 JSON Schema에 맞도록 구조 강제
Prompt내 PowerShell에서 실행할 한 줄 명령을 만들어 줘. 이 학생 명단 CSV를 stdin으로 claude -p에 흘려보내서 "학생 명단 헤더를 본인 표준 컬럼명으로 통일하고 결측은 NA"로 변환, 결과를 data\clean.csv로 저장. --max-budget-usd 1.00 캡도. 한 줄로.
야간 평가표 후처리 표준. --max-budget 캡 안전벨트.
A2 Hooks (라이프사이클)
Adv 2
30+ 라이프사이클 이벤트(PreToolUse·PostToolUse·Notification·PreCompact 등)에 핸들러를 붙임.
settings.json hook
// .claude/settings.json "hooks":{ "PreToolUse":[{ // 도구 실행 직전에 가로채는 훅 "matcher":"Bash(Remove-Item *)", // 매처 = 이 패턴의 명령에만 발동 "command":"powershell -c exit 2" // 실행할 명령 (exit 2 = 차단 + 사용자 확인 강제) }] } # exit 2 = 차단
Promptsettings.json에 PowerShell Remove-Item * 명령을 PreToolUse에서 막는 hook 하나 넣어 줘. exit 2로 차단.
치명적 함정 둘. 무한 루프(같은 도구 재호출 금지)·외부 호출(기관 정책 검토). 기관 환경. allowManagedHooksOnly=true.
A3 MCP
Adv 3
오픈 표준 외부 시스템 연결. 학교·학원 자료 저장소·단톡방·드라이브·LMS·학생 DB. 한 채팅에서 끌어와 요약하고 단톡방에 올리기 한 흐름.
PowerShell · Slash
PS> claude mcp add wiki ` # MCP 서버 등록 (이름 = wiki) --transport http --scope project ` # 통신 = http / 적용 범위 = 이 프로젝트만 --env WIKI_TOKEN=$env:WT ` # 서버에 전달할 환경변수 (토큰) -- https://wiki/.../mcp # 실제 MCP 엔드포인트 > /mcp # 현재 세션의 MCP 서버 연결 상태 확인
Prompt학교·학원 자료 저장소에서 본인이 자주 보는 페이지를 가져와서 5줄로 요약해 줘. 그 요약을 단톡방·팀 채널에 바로 올려 줘.
기관망·학내망 사전 검토 필수. 토큰은 환경변수로만 (settings.json 평문 금지).
A4 Plugins
Adv 4
스킬·서브에이전트·훅·MCP·LSP를 한 패키지로. plugin.json + 마켓플레이스. 강의 표준을 동료 교육자 5명에게 한 줄로 배포.
PowerShell
PS> claude plugin install my-class-kit@private-mp # <플러그인>@<마켓플레이스> 형태로 설치 PS> claude plugin validate # plugin.json·필수 파일 유효성 검사 PS> claude plugin list # 현재 설치된 플러그인과 그 안의 스킬·에이전트
Prompt내 ~/.claude/skills/ 랑 ~/.claude/agents/를 묶어서 'my-class-kit' 플러그인의 plugin.json을 작성해 줘.
새 동료 교육자 첫날 install 한 줄로 동일한 환경. 교내·학원 마켓플레이스는 private repo로.
A5 Routines (클라우드 영구 스케줄)
Adv 5 · 베타
Anthropic 클라우드에서 노트북이 꺼져도 도는 영구 작업. cron + API endpoint + GitHub event 3종 트리거 조합. 베타 헤더 experimental-cc-routine-2026-04-01.
Slash · 자연어 cron
> /schedule weekdays 8am, 어제 교육 뉴스·교과 보도자료 5개 요약하고 강의팀 게시판에 게시 # 자연어 cron + 작업 (클라우드 영구) > /schedule list # 등록된 routine 전체 + 다음 실행 시각 > /schedule update <id> # 기존 routine의 일정·지시문 갱신
REST API · trigger

Prompt/loop 10m queue 상태 확인해 주고, 끝났으면 후처리 시작해 줘. 다 되면 내가 Esc로 멈출게.

Slash

Adv 9

Slash · CLI · frontmatter

섹션 06 · 채택 장벽

보안 · 재현성 · 환각 — 채택 장벽 세 가지에 한 줄씩

보안

API 모드는 데이터 학습을 하지 않습니다. 컨슈머 Pro와 다릅니다. 학내망·기관망 망분리에서는 Bedrock·Vertex·Foundry 라우팅 가능. MCP 화이트리스트는 allowedMcpServers·allowManagedMcpServersOnly로.

재현성

permissions.deny + .claude/rules/ paths + git 세 겹으로 변경 추적. raw 데이터는 paths 글롭으로 자동 보호. CLAUDE.md·SKILL.md를 git에 같이 두면 환경 재현 가능.

환각

인용 검수 룰 + Plan 모드 + /ultrareview 다관점 세 가지로 완화. "검수하면 돼요"가 아니라 구체적 도구로 답합니다. 한 사람 검수보다 단단합니다.
섹션 06 · 자기 점검 (LO 1~5)

5문장 자기 점검 (LO 1~5)
